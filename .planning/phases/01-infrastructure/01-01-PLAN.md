<task type="auto">
  <name>Task 2: Enhance RabbitMQConsumer with production patterns</name>
  <files>src/services/rabbitmq_consumer.rs</files>
  <action>
Refactor the consumer to add production patterns:

1. **Replace println!/eprintln! with log macros:**
   - Use `info!` for connection status, message received/acknowledged
   - Use `warn!` for connection errors, retry attempts
   - Use `error!` for message handling failures

2. **Add exponential backoff constants:**
   - INITIAL_RETRY_DELAY_MS: 1000 (1 second)
   - MAX_RETRY_DELAY_MS: 30000 (30 seconds)
   - BACKOFF_MULTIPLIER: 2.0

3. **Add graceful shutdown:**
   - Add `shutdown_tx: tokio::sync::broadcast::Sender<()>` to struct
   - Initialize in constructor: `let (shutdown_tx, _) = broadcast::channel(1);`
   - Use `tokio::select!` to listen for shutdown signal alongside consumer
   - Implement `Drop` trait to call shutdown()

4. **Add DLQ configuration:**
   - On queue_declare, add FieldTable with:
     - "x-dead-letter-exchange" = "obligation_dlx"
     - "x-dead-letter-routing-key" = "failed"

5. **Fix message acknowledgment:**
   - On handler success: `delivery.ack().await?`
   - On handler failure: `delivery.nack(BasicNackOptions { requeue: false }).await?` — sends to DLQ

6. **Add Connection status visibility:**
   - Log "Connected to RabbitMQ" on successful connection
   - Log "Consumer started, waiting for messages" after basic_consume

Reference the pattern from 01-RESEARCH.md lines 95-256.
