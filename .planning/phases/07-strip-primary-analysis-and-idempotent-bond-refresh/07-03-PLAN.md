  <action>
    Заменить тело save_bond на идемпотентный upsert. Подход: SELECT-then-decide,
    чтобы корректно различать created / updated / unchanged и условно писать
    price_history (чистый ON CONFLICT не возвращает «что изменилось»). Шаги внутри
    одной транзакции (pool.begin()):

    1. SELECT текущей строки по ticker (как минимум price и coupon_amount) из
       obligation_crawler_bonds через fetch_optional.

    2. Выполнить upsert: INSERT INTO obligation_crawler_bonds (run_id, ticker, name,
       price, yield_to_maturity, coupon_type, next_coupon, coupon_amount,
       accrued_coupon_income, payments_per_year, maturity, volume, subordinated,
       amortization, for_qualified_investors, change_today, analysis) VALUES ($1..$17)
       ON CONFLICT (ticker) DO UPDATE SET run_id = EXCLUDED.run_id,
       name = EXCLUDED.name, price = EXCLUDED.price,
       yield_to_maturity = EXCLUDED.yield_to_maturity,
       coupon_type = EXCLUDED.coupon_type, next_coupon = EXCLUDED.next_coupon,
       coupon_amount = EXCLUDED.coupon_amount,
       accrued_coupon_income = EXCLUDED.accrued_coupon_income,
       payments_per_year = EXCLUDED.payments_per_year, maturity = EXCLUDED.maturity,
       volume = EXCLUDED.volume, subordinated = EXCLUDED.subordinated,
       amortization = EXCLUDED.amortization,
       for_qualified_investors = EXCLUDED.for_qualified_investors,
       change_today = EXCLUDED.change_today, updated_at = NOW() RETURNING id.
       Намеренно НЕ упоминать created_at в DO UPDATE (остаётся прежним при апдейте)
       и НЕ обновлять analysis (downstream API владеет этой колонкой; затирание
       NULL'ом стёрло бы анализ API).

    3. Определить исход (enum SaveOutcome):
       - предыдущей строки не было (шаг 1 → None) → Created;
       - была и price ИЛИ coupon_amount отличаются от новых значений → Updated +
         INSERT INTO obligation_crawler_price_history (ticker, price, coupon_amount,
         run_id) VALUES (...);
       - была и ни price, ни coupon не изменились → Unchanged (в price_history НЕ
         писать). Сравнение Option<f64> точное (значения парсятся из текста, не
         вычисляются), это приемлемо.

    4. COMMIT. Изменить тип возврата save_bond с Result<Uuid, sqlx::Error> на
       Result<SaveOutcome, sqlx::Error>, где SaveOutcome — новый enum в этом файле
       { Created, Updated, Unchanged } с derive(Debug, Clone, Copy, PartialEq, Eq).
       Это контракт для счётчиков в bonds_crawler (Task 2).

    Конвенции: без unwrap/expect вне тестов, ошибки через `?` (sqlx::Error
    пробрасывается), hard-tabs, English doc-комментарии на изменённых функциях, без
    println! (логирование счётчиков — в вызывающем слое).
