<objective>
Заменить append-only INSERT в репозитории на идемпотентный upsert по ticker:
`INSERT ... ON CONFLICT (ticker) DO UPDATE`. На повторном прогоне обновлять поля,
сохранять created_at и обновлять updated_at, а при изменении цены (и/или купона)
дописывать строку в obligation_crawler_price_history. По итогам прогона логировать
created / updated / unchanged.

Purpose: Устранить дублирование облигаций между прогонами (корневой баг из
CONCERNS.md "Duplicate bond accumulation"). Краулер становится идемпотентным.
Output: Обновлённый bonds_repository.rs (upsert + price_history + счётчики) и
скорректированный вызов в bonds_crawler.rs с агрегацией результата прогона.

Важно про видимость mid-run: upsert перепривязывает `run_id` строки к текущему
('running') прогону. Это безопасно, потому что план 02 СНЯЛ фильтр
`status='completed'` из вью obligation_crawler_latest_bonds — бумага остаётся видна
API во время прогона. Здесь, в репозитории, дополнительных действий по видимости НЕ
требуется: `run_id = EXCLUDED.run_id` в DO UPDATE — корректно.
</objective>
