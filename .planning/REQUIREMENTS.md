# Requirements — V1

**Project:** obligation_crawler

##REQ-001: RabbitMQ Consumer

Слушает RabbitMQ очередь и запускает парсинг по триггеру.

- Подключается к RabbitMQ (lapin)
- Слушает очередь задач (по образцу существующего сервиса)
- При получении задачи запускает парсинг облигаций
- Обрабатывает ошибки подключения с retry логикой

**Acceptance:**
- [ ] Подключение к RabbitMQ работает
- [ ] Сообщения из очереди обрабатываются
- [ ] Запускается парсинг при получении задачи

---

##REQ-002: T-Bank Bonds Parser — Фаза 1 (Quick List)

Быстрый прогон по списку облигаций без открытия вкладок.

- Открывает страницу списка облигаций T-Bank
- Извлекает: ticker, name, price (без детального парсинга)
- Не открывает каждую облигацию в отдельной вкладке
- Результат: preliminary bond list

**Acceptance:**
- [ ] Парсит список облигаций со страницы
- [ ] Извлекает ticker, name, price для каждой облигации
- [ ] Не открывает детальные вкладки (быстро)

---

##REQ-003: T-Bank Bonds Parser — Фаза 2 (Detailed)

Детальный прогон — открывает каждую облигацию для получения полных данных.

- Принимает список из фазы 1
- Открывает каждую облигацию в отдельной вкладке
- Извлекает все поля: yield, coupon_type, next_coupon, maturity, volume, etc.
- Сохраняет полные данные облигации

**Acceptance:**
- [ ] Открывает детальную страницу каждой облигации
- [ ] Извлекает все поля схемы БД
- [ ] Обрабатывает ошибки (недоступные облигации)

---

##REQ-004: PostgreSQL Write

Записывает результаты парсинга в PostgreSQL.

- Использует существующую схему (`001_create_crawler_schema.sql`)
- Записывает в `obligation_crawler_runs` — сессия краулинга
- Записывает в `obligation_crawler_bonds` — данные облигаций
- Обрабатывает ошибки БД (retry, fallback)

**Acceptance:**
- [ ] Записывает run в `obligation_crawler_runs`
- [ ] Записывает bonds в `obligation_crawler_bonds`
- [ ] Обрабатывает ошибки подключения

---

##REQ-005: RabbitMQ Producer — Completion Notification

Публикует уведомление о завершении парсинга.

- По образцу `analytics_publisher.rs`
- Формат сообщения аналогичен существующему
- Routing key: уведомление о завершении
- Exchange: avito_exchange (или аналог)

**Acceptance:**
- [ ] Публикует сообщение в RabbitMQ при завершении
- [ ] Формат сообщения соответствует образцу
- [ ] Обрабатывает ошибки отправки

---

##REQ-006: Crawler Run Lifecycle

Управление жизненным циклом запуска парсинга.

- Статусы: running → completed | failed
- Записывает start time, end time, duration
- Записывает количество обработанных облигаций
- Записывает error_message при неудаче

**Acceptance:**
- [ ] Статус корректно меняется при start/complete/error
- [ ] Duration вычисляется правильно
- [ ] error_message записывается при ошибках

---

##REQ-007: Basic Metrics

Базовые метрики выполнения парсинга.

- Логирование начала/конца парсинга
- Количество обработанных облигаций
- Ошибки парсинга (сколько не удалось распарсить)
- Duration каждой фазы

**Acceptance:**
- [ ] Логи содержат ключевые события
- [ ] Метрики записываются в БД
- [ ] Можно отследить успешность запусков
