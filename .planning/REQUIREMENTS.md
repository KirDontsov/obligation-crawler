# Requirements — V1

**Project:** obligation_crawler

##REQ-001: RabbitMQ Consumer

Слушает RabbitMQ очередь и запускает парсинг по триггеру.

- Подключается к RabbitMQ (lapin)
- Слушает очередь задач (по образцу существующего сервиса)
- При получении задачи запускает парсинг облигаций
- Обрабатывает ошибки подключения с retry логикой

**Acceptance:**
- [ ] Подключение к RabbitMQ работает
- [ ] Сообщения из очереди обрабатываются
- [ ] Запускается парсинг при получении задачи

---

##REQ-002: T-Bank Bonds Parser — Фаза 1 (Quick List)

Быстрый прогон по списку облигаций без открытия вкладок.

- Открывает страницу списка облигаций T-Bank
- Извлекает: ticker, name, price (без детального парсинга)
- Не открывает каждую облигацию в отдельной вкладке
- Результат: preliminary bond list

**Acceptance:**
- [ ] Парсит список облигаций со страницы
- [ ] Извлекает ticker, name, price для каждой облигации
- [ ] Не открывает детальные вкладки (быстро)

---

##REQ-003: T-Bank Bonds Parser — Фаза 2 (Detailed)

Детальный прогон — открывает каждую облигацию для получения полных данных.

- Принимает список из фазы 1
- Открывает каждую облигацию в отдельной вкладке
- Извлекает все поля: yield, coupon_type, next_coupon, maturity, volume, etc.
- Сохраняет полные данные облигации

**Acceptance:**
- [ ] Открывает детальную страницу каждой облигации
- [ ] Извлекает все поля схемы БД
- [ ] Обрабатывает ошибки (недоступные облигации)

---

##REQ-004: PostgreSQL Write

Записывает результаты парсинга в PostgreSQL.

- Использует существующую схему (`001_create_crawler_schema.sql`)
- Записывает в `obligation_crawler_runs` — сессия краулинга
- Записывает в `obligation_crawler_bonds` — данные облигаций
- Обрабатывает ошибки БД (retry, fallback)

**Acceptance:**
- [ ] Записывает run в `obligation_crawler_runs`
- [ ] Записывает bonds в `obligation_crawler_bonds`
- [ ] Обрабатывает ошибки подключения

## V2 (Phase 7)

##REQ-008: Idempotent Bond Refresh (no duplication)

Повторный прогон обновляет существующие облигации вместо создания дубликатов.

- Идентификатор бумаги между прогонами — `ticker` (ISIN краулер не собирает)
- На прогоне: найти существующую бумагу по тикеру, сравнить распарсенные поля с сохранёнными
- Обновлять только изменившиеся поля (чаще всего `price`, возможно `coupon_amount`, `accrued_coupon_income`, `yield_to_maturity`)
- Модель данных: текущее состояние (одна строка на тикер, upsert in place) + отдельная лёгкая таблица `obligation_crawler_price_history` (строка добавляется только при изменении цены/купона)
- Read-контракт downstream API сохраняется: семантика "latest per ticker" продолжает работать без изменений в API

**Acceptance:**
- [ ] Повторный прогон не создаёт дубликатов (одна строка на тикер)
- [ ] Изменение цены обновляет строку bond и добавляет запись в price_history
- [ ] Неизменные поля не перезаписываются без необходимости
- [ ] API читает данные без изменений своего кода

---

##REQ-009: Remove Primary Analysis from Crawler

Краулер становится чистым scrape+parse+persist; AI-анализ удаляется.

- Удалить интеграцию с opencode (`src/services/opencode_service.rs` и вызов в `bonds_crawler.rs`)
- Удалить skip-analysis логику
- Колонка `analysis` остаётся в схеме (её будет заполнять downstream API), но краулер её больше не пишет
- Ответственность за первичный анализ полностью переходит к obligation-api (реализуется отдельной фазой в том репозитории)

**Acceptance:**
- [ ] В краулере не осталось вызовов opencode
- [ ] Краулер собирается и проходит прогон без анализа
- [ ] Поле `analysis` в записях краулера = NULL (не заполняется)
